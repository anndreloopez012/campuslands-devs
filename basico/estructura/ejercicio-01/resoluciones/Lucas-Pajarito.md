# Estructura de archivos

En este ejercicio practicamos la organizacion de archivos al momento de trabajar en proyecto ya sea grandes o pequeños. 
```bash
resoluciones/
│
├── assets/
│   ├── audio/
|   |   └── .gitkeep
│   ├── img/
│   │   └── .gitkeep
│   └── maps/
│       └── .gitkeep
│
├── docs/
│   └── .gitkeep
│
├── src/
│   ├── css/
│   │   └── style.css
│   │
│   ├── html/
│   │   └── index.html
│   │
│   └── js/
│       └── main.js
│
├── tests/
│   └── .gitkeep
│
├── .gitkeep
│
└── Lucas-Pajarito.md
```

# Explicación de cada carpeta

1.  <strong>Carpeta assets :</strong> En esta carpeta se se crearon otras tres adicionales:<br>


    *  <strong>Imagenes(img):</strong> Dentro de esta carpeta de guardaran archivos visuales para utilizarlos en el proyecto.<br>

    *  <strong>Audio:</strong> Esta carpeta sirve para almacenar archivos de sonido o multimedia que podamos necesitar en nuetro poryecto.

    *  <strong>Maps:</strong> Esta carpeta almacena de ubicación o mapas(Google maps)

2. <strong>Docs</strong>: En una carpeta docs se almacenan archivos o documentos que van relacionados al proyecto como por ejemplo: Manuales, guias diagramas de flujo y notas tecnicas.

3.  <strong>Src:</strong> En las carpetas Src se guardan los archichos fuentes como por ejemplo: HTML, CSS, JavaScript, etc.

4.  <strong>Tests:</strong>  Su función principal es almacenar archivos de prueba o tests, cuyo objetivo es verificar que el código funcione correctamente.


### ¿Que es .gitkeep?
  GitKeep no es un comando oficial de Git ni una herramienta independiente; más bien, es un concepto o práctica que se utiliza al trabajar con repositorios Git para manejar carpetas vacías.

# 📁 Estructura del Proyecto

El proyecto está organizado de manera modular para separar correctamente los recursos, documentación, código fuente y pruebas.

```plaintext
.
├── assets/
│   ├── audio/
│   │   └── .gitkeep
│   ├── images/
│   │   └── .gitkeep
│   └── maps/
│       └── .gitkeep
├── docs/
│   └── .gitkeep
├── src/
│   └── .gitkeep
├── test/
│   └── .gitkeep
└── wilder-catu.md
```

---

#  Descripción de Carpetas

## `assets/`

Contiene todos los recursos estáticos utilizados en el proyecto.

### `audio/`

Almacena archivos de sonido, música o efectos de audio.

### `images/`

Guarda imágenes, íconos y recursos gráficos.

### `maps/`

Contiene recursos relacionados con mapas o escenarios.

---

## `docs/`

Incluye la documentación del proyecto, manuales y archivos de apoyo.

---

## `src/`

Contiene el código fuente principal de la aplicación o sistema.

---

## `test/`

Carpeta destinada a pruebas, validaciones y testing del proyecto.

---

#  Nota

Las carpetas vacías incluyen un archivo `.gitkeep` para que Git pueda mantener la estructura de directorios dentro del repositorio, incluso cuando aún no contienen archivos.

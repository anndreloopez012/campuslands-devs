## 📁 Estructura del Proyecto
He diseñado esta arquitectura para mantener un entorno de desarrollo escalable, separando claramente la lógica de los recursos multimedia:

- **`src/`** ✨  
  Contiene toda la lógica central y el código fuente del videojuego.

- **`assets/`** 🎨  
  Recursos multimedia organizados para un flujo de trabajo eficiente:
    - `images/` 🖼️: Sprites, texturas y assets visuales.
    - `audio/` 🎵: Efectos de sonido y pistas musicales.
    - `maps/` 🗺️: Archivos de diseño de niveles y mapas.

- **`docs/`** 📄  
  Documentación técnica y guías del proyecto.

- **`tests/`** ✅  
  Espacio dedicado a scripts de prueba para garantizar la calidad del código.

## 🛠️ Razonamiento Técnico
Para resolver este ejercicio, seguí un enfoque metódico para asegurar el orden y la persistencia en el control de versiones:

1. **Análisis:** Clasifiqué los archivos según su propósito para evitar la desorganización a medida que el proyecto crezca.
2. **Ejecución:** Utilicé comandos granulares en la terminal para crear la estructura de directorios, manteniendo los nombres en minúsculas.
3. **Mantenimiento:** Implementé archivos `.gitkeep` en cada carpeta vacía, asegurando que la jerarquía se mantenga dentro del repositorio Git sin errores.
